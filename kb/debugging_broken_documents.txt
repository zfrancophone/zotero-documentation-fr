====== Dépannage des erreurs dans les documents de traitement de texte ======

Si vous pouvez insérer des citations Zotero dans de nouveaux documents mais que vous recevez une erreur dans un document existant, suivez les étapes suivantes pour trouver la cause de l'erreur.

    - Assurez-vous que vous utilisez les dernières versions de Zotero et de votre traitement de texte.
    - Lors du dépannage, désactivez la fonction de suivi des modifications dans Word/LibreOffice, car elle peut avoir des effets compliqués lorsque vous travaillez avec Zotero. Si la fonction de suivi des modifications est activée lorsque vous insérez ou modifiez une citation Zotero, elle peut marquer plusieurs ou toutes les citations Zotero de votre document comme étant modifiées ou provoquer l'affichage des codes de champ (voir point 9 ci-dessous). En de rares occasions, le suivi des modifications peut amener Zotero à penser qu'une citation est corrompue. Si vous aviez activé le suivi des modifications auparavant, essayez d'accepter toutes les modifications pour voir si cela résout le problème.
    - Vérifiez les citations dans les légendes des images. Zotero ne vous laissera pas les insérer, mais si vous avez copié une citation dans une légende, c'est très probablement la source du problème. Supprimez-la.
    - Essayez de copier et de coller le contenu du document dans un nouveau document pour voir si le problème disparaît. Vous devrez peut-être cliquer sur le bouton "Document Preferences" avant que vos anciennes citations ne soient reconnues.
    - Faites une copie de votre document - en dupliquant le fichier lui-même, et non en copiant et collant le contenu - à utiliser pour le dépannage.
    -  Si vous utilisez OneDrive sur Windows, enregistrez la copie du document sur votre disque dur local. OneDrive est connu pour interférer avec le module pour certaines personnes. Si cela résout le problème pour vous, vous pouvez nous aider à déboguer ce conflit en publiant un message sur les [[/forum|forums Zotero]].
    - Ouvrez le fichier copié et vérifiez si vous obtenez l'erreur après avoir changé de style bibliographique.
    - Si le document comporte une bibliographie, supprimez-la et vérifiez si vous obtenez toujours l'erreur.
    - Lors du dépannage, si vous utilisez le mode "Champs" dans Word ou le mode "Marques de référence" dans LibreOffice, il peut être utile d'afficher les codes de champ plutôt que le texte mis en forme. Pour ce faire, appuyez sur Alt/Option-F9 (ou Alt/Option-Fn-F9) dans Word ou Ctrl-F9 dans LibreOffice.
    - **Isolez les citations problématiques**. Supprimez  une moitié du document d'un seul coup et voyez si l'erreur se produit toujours. Si ce n'est pas le cas, utilisez la fonction "Annuler" pour restaurer la section supprimée et essayez ensuite de supprimer l'autre moitié. Répétez le processus de réduction de moitié sur la section dans laquelle l'erreur se produit, ou choisissez une section au hasard si elle se produit dans les deux. Continuez ainsi jusqu'à ce que vous trouviez la plus petite section possible, idéalement avec une seule citation, qui doit être présente pour que le problème se produise. Retirez les citations ainsi isolées du document original. Le problème devrait disparaître, à moins qu'il n'y ait plusieurs citations cassées, auquel cas vous devrez répéter le processus. À moins que l'erreur ne se produise encore si vous effacez complètement le contenu du document, **cette dernière étape permettra par définition d'identifier le problème.**


Si vous vous heurtez à un document endommagé, veuillez créer un nouveau fil de discussion dans les [[/forum|forums Zotero]] afin que nous puissions tenter de résoudre le problème. Veillez à inclure un [[:reporting_problems#provide_a_report_id|Report ID]] de Zotero, la version de votre système d'exploitation et de votre traitement de texte, les étapes que vous avez suivies pour tenter de corriger l'erreur et si vous avez réussi. Vous devez également envoyer l'extrait de document de l'étape 10 et un lien vers votre fil de discussion sur le forum à [[mailto:support@zotero.org|support@zotero.org]] afin que nous puissions l'examiner.
